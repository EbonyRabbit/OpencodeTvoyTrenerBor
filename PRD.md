# Product Requirements Document: ТвойТренерБот

## 1. О продукте

**ТвойТренерБот** — это Telegram-бот для онлайн-фитнес-коучинга. Клиент покупает программу тренировок и ведёт её через бота: получает тренировки, логирует выполнение, делает замеры, отправляет фото, проходит чек-ины. Бот автоматизирует рутину, напоминает о тренировках и собирает данные о прогрессе.

Продукт уже работает как прототип на стеке Google Apps Script + Google Sheets + Cloudflare Worker.

Целевая аудитория — B2C: люди, которые хотят заниматься по готовым программам с автоматическим сопровождением.

Модель — гибрид: автоматика для массовых клиентов + возможность прямого ведения тренером для премиум-клиентов.

---

## 2. Технологический стек

**Текущий (остаётся):**
- Google Apps Script — логика бота (Code.gs)
- Google Sheets — хранение данных, программы
- Cloudflare Worker — прокси для Telegram webhook
- Telegram Bot API — интерфейс для клиентов
- Python — генерация Excel-программ

**Новый (добавляется):**
- Next.js — веб-панель для тренера
- Supabase (PostgreSQL) — база данных для веб-панели
- TypeScript — язык разработки веб-панели

Архитектура хранения данных — дуальная: бот продолжает писать в Google Sheets, и одновременно дублирует данные в Supabase. Веб-панель работает с Supabase. Это обеспечивает обратную совместимость и возможность постепенного переезда.

---

## 3. Пользовательские роли

### Клиент
- Взаимодействует через Telegram-бота
- Покупает и активирует программы
- Выполняет тренировки, логирует результаты
- Делает замеры и загружает фото
- Проходит еженедельные чек-ины
- Настраивает уведомления

### Тренер/Администратор
- Работает через веб-панель (Next.js) и Google Sheets
- Управляет клиентами (активация, коды, отключение)
- Создаёт программы тренировок (через Python + Sheets)
- Просматривает прогресс клиентов
- Видит алерты и историю
- Общается с клиентами через чат

## 4. Функциональные требования

### 4.1 Telegram-бот (ядро)

#### 4.1.1 Подключение и регистрация
- `/start` — умное приветствие: новому клиенту предлагает ввести код подключения или купить программу, активному показывает меню, неактивному — reconnect
- `/menu` — список доступных команд (адаптируется под статус клиента)
- Поддержка нескольких клиентов на одном Telegram-аккаунте через переключение по коду
- Очистка старого telegram_id при переподключении
- **connect_code** — 8-символьный буквенно-цифровой (например: `TRN-8A2F`), генерируется автоматически при создании личной таблицы клиента

#### 4.1.2 Каталог программ и продажа
- `/programs` — каталог программ с названием, описанием, инвентарём, ценой
- `/myprogram` — информация о текущей программе, ссылка на таблицу, код подключения, оставшееся количество недель доступа
- `/switchprogram` — запрос смены программы. После подтверждения тренером старая таблица архивируется, создаётся новая, доступ перезапускается с W1
- Регистрация нового клиента: выбирает программу → вводит имя → статус `payment_status=pending`
- Оплата — ручная (тренер подтверждает в Google Sheets, меняет `payment_status=paid`)
- После оплаты бот копирует шаблон программы, создаёт личную таблицу клиента, генерирует код и отправляет доступ
- **Доступ по времени** — привязан к длительности программы (например, 12 недель). По истечении срока клиент автоматически блокируется (статус `access_expired`). Продление — только через тренера.
- Пока `payment_status ≠ paid` — команды `/today`, `/progress`, `/checkin` заблокированы

#### 4.1.3 Тренировки
- `/today` — показывает тренировку на сегодня по дню недели (не по дате)
- Если сегодня тренировки нет — показывает ближайшую или определяет день как восстановление
- **Определение недели программы** — от даты старта клиента (день оплаты/активации = День 1, W1 = первые 7 дней). Не привязано к календарным датам.
- Пошаговое логирование упражнения: подходы → повторения → вес → RPE → комментарий
- Данные записываются в недельный лист Google Sheets (факт-колонки) и в сырой лог
- Кнопка «Пропустить тренировку» с опциональным указанием причины
- Автоматический переход к следующему упражнению после логирования

#### 4.1.4 Замеры тела
- `/progress` — меню: начать замеры / посмотреть динамику
- Сбор замеров: вес, талия, живот, грудь, бёдра, ягодицы, левое/правое бедро, левая/правая рука, % жира, мышечная масса, висцеральный жир, комментарий
- После замеров — запрос фото: спереди, сбоку, сзади (с возможностью пропустить)
- Фото сохраняются в Google Drive: корневая папка → папка клиента → папка недели
- Ссылки на фото записываются в личную таблицу клиента

#### 4.1.5 Еженедельный чек-ин
- `/checkin` — 7 шагов: самочувствие → сон → стресс → adherence питания → пропущенные тренировки → жалобы → комментарий
- Результаты сохраняются в лист Check-ins

#### 4.1.6 Напоминания и уведомления
- Утренняя рассылка тренировки в настроенное время (окно 2 часа от `morning_time`)
- Напоминание о замерах в настроенный день/время (до 2 повторных напоминаний с интервалом 2 часа)
- Отложенное напоминание о замерах — запись в `Bot Schedule`, проверка каждые 15 мин
- Вечерний опрос (20:00–00:00): «Ты сегодня тренировался?» — с опциями «Да», «Пропустить», «Перенести»
- Механизм проверки: Google Apps Script time-driven trigger, запуск раз в 15 минут (функция `sendDueMessages()`)
- Флаги отправки — в `PropertiesService` (Script Properties), ключ по шаблону `{type}_{clientId}_{date}`
- Фильтр: уведомления только для активных клиентов с paid-статусом и действующим доступом
- При 500+ клиентах возможно увеличение нагрузки — мониторить лимиты Apps Script (6 мин макс. выполнения)

#### 4.1.7 Настройки
- `/settings` — изменение времени тренировки, времени замеров, дня замеров, часового пояса
- Все настройки сохраняются в Bot Clients

### 4.2 Типы программ

бот поддерживает программы с недельной структурой (W1–W12):

- **Сила/Гипертрофия** — классический сплит (PPL, upper/lower, фулбоди), 8 колонок: Блок, Упражнение, Подходы, Повторы, Вес/%1ПМ, RPE, Отдых, Заметки
- **HYROX подготовка** — бег + силовые станции, 13 колонок: Раздел, Упражнение, План (подходы, повторы, вес, RPE/RIR, темп, отдых) + Факт (подходы, повторы, вес, RPE, комментарий)
- **Домашние тренировки** — с минимальным инвентарём (гантели, резинки)
- **Функциональный тренинг** — CrossFit-style, WOD
- **Кардио/беговые** — беговые программы
- **Детские тренировки (от 9 лет)** — адаптированные программы для детей

Программы создаются через Python-скрипты генерации Excel (.xlsx) с последующей загрузкой шаблона в Google Sheets. Веб-панель в будущем добавит конструктор программ.

### 4.3 Веб-панель тренера (Next.js + Supabase)

#### 4.3.1 Дашборд
- Сводка по всем клиентам: статус, adherence, пропуски, прогресс
- Графики и визуализация ключевых метрик
- Алерты: низкая adherence, пропуски, высокий стресс, жалобы

#### 4.3.2 Управление клиентами
- Список всех клиентов с фильтрацией и поиском
- Просмотр профиля клиента: программы, прогресс, история
- Ручные действия: активация программы, генерация кода, отключение
- Изменение статуса оплаты

#### 4.3.3 Прогресс и аналитика
- Графики веса, замеров, % жира по неделям
- История чек-инов с визуализацией трендов
- Adherence по тренировкам (% выполненных)
- Динамика рабочих весов по упражнениям

#### 4.3.4 Управление программами
- Каталог программ: создание, редактирование, публикация
- Загрузка шаблонов программ
- Назначение программы клиенту

#### 4.3.5 Чат с клиентом
- Тренер пишет сообщение в веб-панели → бот отправляет его клиенту в Telegram
- Клиент отвечает в Telegram → ответ отображается в веб-панели в истории чата
- Клиент может инициировать диалог командой `/chat`
- История сообщений хранится в таблице `messages` в Supabase

#### 4.3.6 Уведомления
- Просмотр истории уведомлений
- Ручная отправка рассылки
- Алерты на email/Telegram тренеру

### 4.4 Локализация

- Поддержка русского и английского языков
- Выбор языка при регистрации или в настройках
- Все сообщения бота и интерфейс панели переведены

## 5. Данные

### 5.1 Supabase — основные таблицы

- **clients** — id, telegram_id, name, status (active/inactive/access_expired), payment_status (pending/paid), program_id, connect_code, spreadsheet_id, language, timezone, morning_time, measurement_time, measurement_day, access_start_date, access_end_date
- **programs** — id, title, description, equipment, price, template_id, active, type, language, duration_weeks
- **workout_logs** — id, client_id, date, week, exercise, sets, reps, weight, rpe, comment
- **measurements** — id, client_id, date, weight, waist, abdomen, chest, hips, glutes, left_thigh, right_thigh, left_arm, right_arm, body_fat, muscle_mass, visceral_fat, comment
- **checkins** — id, client_id, date, week, wellbeing, sleep, stress, nutrition_adherence, missed_workouts, complaints, comment
- **photos** — id, client_id, date, week, type (front/side/back), drive_url, folder_url
- **program_schedule** — id, client_id, week_number, sheet_name, start_date, end_date, focus, status
- **exercises** — id, name, muscle_group, equipment, difficulty, demo_video_url, contraindications
- **messages** — id, client_id, coach_id, direction (to_client/to_coach), text, sent_at, read_at
- **notification_log** — id, client_id, type, status, sent_at, metadata

### 5.1.1 Связи таблиц

- `clients.program_id` → `programs.id`
- `workout_logs.client_id` → `clients.id`
- `measurements.client_id` → `clients.id`
- `checkins.client_id` → `clients.id`
- `photos.client_id` → `clients.id`
- `program_schedule.client_id` → `clients.id`
- `messages.client_id` → `clients.id`
- `notification_log.client_id` → `clients.id`

Длительность доступа клиента определяется полем `programs.duration_weeks`. По истечении срока от даты `clients.created_at` клиент переводится в статус `access_expired`.

Все таблицы содержат created_at и updated_at.

### 5.2 Синхронизация

Бот при каждом действии пишет данные в Google Sheets и в Supabase (через API). Веб-панель читает исключительно из Supabase. При переезде на Node.js Google Sheets остаётся как бэкап и для обратной совместимости.

## 6. Roadmap

### Этап 1: Веб-панель + Supabase
- Настроить Supabase, создать схему БД
- Разработать Next.js веб-панель: дашборд, список клиентов, чек-ины, прогресс
- Добавить дуальную запись в бота (Sheets + Supabase API)
- Миграция существующих данных в Supabase

### Этап 2: История и статистика
- Детальная аналитика в боте и панели
- Графики динамики весов, замеров, adherence
- История тренировок, сравнение циклов

### Этап 3: AI-функции
- AI-анализ чек-инов с рекомендациями
- AI-чат с клиентом (ответы на вопросы)
- AI-генерация программ по промпту

### Этап 4: Платежи онлайн
- Интеграция Telegram Stars
- Stripe / ЮKassa для внешних платежей
- Автоматическая активация при оплате

### Этап 5: Интеграции
- Apple Health / Google Fit
- Носимые устройства
- MyFitnessPal

### Этап 6: Переезд бота на Node.js
- Замена Apps Script на Node.js/TypeScript
- Единый стек: Next.js + Node.js + Supabase
- Google Sheets как бэкап, не как источник правды

## 7. UI/UX принципы

### Тон коммуникации
- Профессиональный фитнес-эксперт — в рекомендациях и программах
- Мотивирующий друг — в напоминаниях и прогрессе
- Игровая механика — в достижениях, стриках, рекордах

### Telegram-бот
- Минималистичные сообщения, кнопки вместо ввода
- Пошаговый ввод — одно поле за раз
- Мгновенная обратная связь на каждое действие

### Веб-панель
- Адаптивный дизайн (desktop + mobile browser)
- Одноэкранные дашборды, минимум кликов
- Визуализация данных — графики, прогресс-бары, цветовые индикаторы

## 8. Ограничения и допущения

- AI-функции не входят в MVP
- Оплата остаётся ручной до Этапа 4
- Интеграция с wearables не входит в MVP
- Бот остаётся на Apps Script до Этапа 6
- Веб-панель (MVP) работает без авторизации — доступ по прямой ссылке
- Детские тренировки не продаются через бота — только через тренера напрямую
- Программы создаются через Python + Excel (не в веб-панели)
- Адаптация программ под клиента не предусмотрена (фиксированные программы)
- Целевая нагрузка — 500+ клиентов на одного бота

---

## 9. Критерии успеха MVP

- Клиент может купить программу, подключиться и начать тренироваться без участия тренера (кроме подтверждения оплаты)
- Клиент получает утренние напоминания о тренировках
- Клиент логирует тренировки, замеры и фото
- Клиент проходит еженедельный чек-ин
- Тренер видит всех клиентов, их прогресс и чек-ины в веб-панели
- Тренер может активировать/деактивировать клиентов через панель
- Поддержка русского и английского языков
